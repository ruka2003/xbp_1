gitとは何か

ファイルの変更履歴を管理してくれるシステムのひとつ

＜特徴＞

ローカル環境にもコードの変更履歴を保存（コミット）することができ
ネットワークに接続していなくても作業を行うことができる

  • 分散型バージョン管理システム
    • 中心リポジトリのコピーをローカル環境に作り出す
    これをローカルリポジトリという
  • CUI使用⇔GUI
    • キーボード使用⇔マウスやクリックで直接的に使われる

  • 安全性が高い
    • 暗号化されたパスワード使用しているため、安全に守られている

  • 柔軟性が高い
    • 多方面で利用できる

    ＜用語集＞

    Clone
    リモートリポジトリを複製してローカルリポジトリを作ること

    Repository
    履歴管理を行う場所

    Push
    ローカルリポジトリの変更をリモートリポジトリに反映させること

    Branch
    履歴管理を枝のようにわかれさせたもの

    Commit
    インデックスに登録してある変更対象をローカルリポジトリに反映させること

    Merge
    異なるブランチの変更を反映すること

    Pull
    リモートリポジトリの変更をローカルリポジトリに反映させること

    Fetch
    リモートリポジトリの変更をローカルリポジトリに反映させること

    インデックス
    コミットしたいファイルやファイルの一部を登録する部分

    ワークツリー
    履歴管理を行いたいファイルがある場所

    HTMLとは何か


    Text　Markup　Languageの略語

    ―解説―


    ウェブ上で公開しているページのテキストやレイアウトなどの構造を定義をしている書式のことを言う
    HTMLは「マークアップ言語」に分類される（文章に意味づけをする）

    ＜HTMLの誕生までの流れ＞

    １９８０年代にSGMLが誕生し、多くの人が簡単に利用できるようにするため、作られたものがHTMLである

    見た目の変更は「css」を使う


    ＜違い＞


    HTML4.01…少し前のウェブサイト上のサイト

    XHTML1.0…記述ルールが厳密
    他と書き方が異なる部分が多い

    HTML5…HTML内の最新バージョン
    　　　　　モダンなサイトである

    4つにファイルが分類される


  • ファイル　＜内容＞

  • CSSファイル　＜見た目＞

  • 画像ファイル　＜絵＞

  • ファイル　＜動作＞
